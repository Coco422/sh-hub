# AGENTS.md

本文件约束这个仓库里的所有贡献者与 AI agent。若与其他习惯冲突，以本文件为准。

## 仓库定位

这是一个通用 Shell 脚本 hub，脚本会被拉到不同服务器上直接使用。

这里的最高优先级是：

1. 不泄露敏感信息
2. 保持脚本简单、可审查
3. 保持通用、稳定、易落地

## 强制规则

### 1. 仓库里永远不能出现任何敏感信息

禁止提交、硬编码或示例化以下内容：

- 密码、Token、API Key、私钥、证书、Cookie、SSH 凭据
- 真实服务器 IP、真实域名、内网地址、内部路径
- 客户数据、账号信息、连接串、配置密文
- 任何其他不能公开放进 Git 仓库的内容

执行要求：

- 敏感输入只能在运行时通过交互输入、环境变量或本地未跟踪文件提供
- 仓库内只允许使用占位符，例如 `YOUR_TOKEN`、`example.com`、`127.0.0.1`
- 脚本、README、注释、示例、提交说明里都不能出现敏感信息
- 如果发现疑似敏感信息，先移除或替换，再继续提交

### 2. 脚本必须坚持 KISS 原则

这个仓库的脚本要以“能快速人工审查”为第一目标，不追求花哨写法。

执行要求：

- 一个脚本只解决一个明确问题
- 优先使用直白、顺序化、容易读懂的 Shell 写法
- 不做过度抽象，不堆框架，不写炫技式逻辑
- 新增依赖要非常克制，能不用就不用
- 如果一段逻辑需要很长解释，优先继续简化，而不是补更多注释

## 编写约定

- 默认显式声明 shebang；使用 Bash 时优先 `#!/usr/bin/env bash`
- 失败要尽早暴露，必要时检查权限、输入和依赖
- 涉及高风险操作时，必须给出清晰提示；必要时要求确认
- 输出要让人看得懂当前要做什么、正在做什么、做完了什么
- 任何改动都应优先提升可读性、可维护性和可审查性

## 提交前自检

提交任何脚本或文档前，先确认：

1. 没有任何敏感信息进入仓库
2. 脚本没有被写复杂
3. 没有为了复用或“优雅”牺牲可审查性
4. 其他人拿到后能快速看懂并放心执行
